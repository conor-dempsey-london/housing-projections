**Prerequisite check:** `pixi run test` (472 passed, 23 slow tests deselected) / `pixi run lint` (all checks passed) / `pixi run typecheck` (no issues, 23 source files) — all pass.

**Summary:** The package is in good shape overall — no circular imports, no junk-drawer modules, no speculative abstractions, clean computation/I/O separation in `data.py`, and a `DwellingModel` ABC that's a textbook-legitimate use of inheritance across 29 subclasses that genuinely share a fit/sample/save lifecycle. The most consequential finding is the silent, undocumented `except Exception: pass` cluster in `html_report.py`'s `_build_eda` (lines 288-341), which is inconsistent with the logged/commented exception handling used everywhere else in the same file and could hide real failures in the generated report. Behind that: the top-level `__init__.py`'s 32-name re-export inconsistently excludes `multimodality` and `eda` without an apparent reason (both are as public-facing as `sensitivity`/`diagnostics`, which are re-exported); `reporting.py`'s `full_report` and `html_report.py`'s `generate_report` are two similarly-named, independently-maintained "build a report" functions with no documented distinction — and only the second one is what the CLI and current workflow actually use, which is a real "which one is current" trap for a new contributor; and test coverage has a gap around `analysis.py` and `html_report.py`, both of which contain non-trivial logic exercised only indirectly (if at all) by the existing suite. `models/models.py` at ~5,000 lines is large but single-purpose and was already de-shotgunned by the `ALL_MODELS` registry pattern, which is worth protecting as new `AZ*` variants continue to be added. None of these are urgent; they're the kind of findings worth picking up opportunistically alongside the next `AZ*` round rather than as a dedicated cleanup pass.
